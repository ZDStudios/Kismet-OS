Vision

A calmer, sharper take on the AI desktop.

Kismet OS is not pretending to be a finished autonomous machine overlord. The point is to build a desktop Linux environment that already feels better on day one, then add deeper intelligence in ways that stay useful, inspectable, and sane.

Local-first by default

Local models and on-device workflows should be first-class, not an afterthought glued on after install.

Developer-ready

A proper terminal, sensible packages, container workflow support, and fewer hours wasted rebuilding the same machine again.

Honest scope

The vision can be ambitious. The repo should still tell the truth about what exists today and what is still ahead.

The long game is intelligence woven into the environment. The short game is making the environment worth using before the intelligence gets clever.
Architecture

Start with a strong system layer, not marketing fog.

The first real version of Kismet OS should be an opinionated Linux environment with coherent theming, a serious developer stack, and a disciplined local AI layer. Reinventing the kernel can wait until there is something worth booting every day.

Planned stack

Base distro with a clear maintenance strategy and clean upgrade path
KDE Plasma desktop plus branded login, splash, and boot flow
Developer package baseline with zsh, Docker, Node, Python, and tooling
Ollama and optional OpenClaw integrations for local AI workflows
A future control layer for services, permissions, updates, and model management

Design rule

If a feature sounds impressive but cannot be implemented safely, inspected easily, or explained clearly, it probably does not belong in the first release.

desktop/ plasma theme, login, splash, boot visuals
system/ services, helpers, post-install automation
ai/ local models, orchestration, safe integrations
docs/ architecture, roadmap, packaging notes
