# Client requirements, distilled

This document translates the submitted custom Linux OS form into build-driving requirements.

## High-priority requirements
- Ubuntu-friendly practical base direction, while keeping room for stronger enthusiast polish
- KDE Plasma desktop
- SDDM display manager with themed login screen
- themed boot flow and Plymouth splash
- graphical installer direction
- zsh default shell
- Kitty or Alacritty style terminal preference
- developer environment preinstalled
- Docker usable without sudo friction
- minimal install, avoid unnecessary desktop clutter
- OpenClaw, Claude Code style workflows, and Ollama integration direction

## Design interpretation
- Zorin OS influence for polish, onboarding, and coherence
- Garuda Linux influence for stronger enthusiast confidence and out-of-box energy
- keep the result premium, sharp, and not gaudy

## Reality discipline
Some requests in the original form describe the ambition, not the current implementation. The build should keep moving toward those goals without pretending they already ship.
